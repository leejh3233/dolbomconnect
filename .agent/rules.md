# Dolbom Connect Project Rules & Standards

이 문서는 프로젝트의 '완벽한' 상태를 유지하기 위한 핵심 기술 표준과 비즈니스 용어 규칙을 담고 있습니다. 향후 모든 작업 시 이 규칙을 최우선으로 준수해야 합니다.

## 1. 용어 준수 규칙 (Terminology)
*   **표준 용어**: `외부파트너` (External Partner)
*   **금지 용어**: `인플루언서` (Influencer)
*   **사유**: 브랜드 정체성 통일 및 파트너십 성격 명확화.
*   **적용 범위**: UI 텍스트, API 필드명(가능한 경우), 주석, 오류 메시지 등 모든 영역.

## 2. 구글 시트 연동 표준 (Google Sheets Robustness)
구글 시트는 사용자가 컬럼 순서를 자유롭게 바꿀 수 있는 환경입니다. 따라서 다음 원칙을 반드시 지킵니다.
*   **동적 헤더 매핑 (Dynamic Header Mapping)**: 고정된 컬럼 인덱스(예: `ph[0]`)를 사용하지 않습니다. 대신 헤더 이름(`이름`, `등록일`, `유형` 등)을 검색하여 인덱스를 찾아내는 로직을 사용합니다.
*   **필수 헤더 검색**: `getColName` 또는 `getColIdx`와 같은 헬퍼 함수를 사용하여 대소문자 구분 없이 유사한 이름의 헤더를 찾아내도록 설계합니다.

## 3. 데이터 기록 원칙
*   **등록일 기록**: 새로운 파트너 등록 또는 데이터 유입 시, 반드시 `등록일` 또는 `CreatedDate` 컬럼을 찾아 현재 시간(KST 기준)을 기록합니다.
*   **자동화**: 날짜 기록은 서버(GAS 또는 API)에서 자동으로 수행하며, 사용자 입력에 의존하지 않습니다.

## 4. 정산 확인 프로세스
*   **직원 vs 파트너**: '직원' 유형의 경우 계좌 정보 없이도 정산 프로세스(급여 합산 안내)가 진행되어야 하며, '외부파트너'는 계좌 정보 유무를 반드시 확인해야 합니다.

---
**기억하세요**: 이 표준은 프로젝트의 안정성과 일관성을 지키는 핵심입니다. 절대 잊지 말고 모든 업데이트 시 적용하십시오.
